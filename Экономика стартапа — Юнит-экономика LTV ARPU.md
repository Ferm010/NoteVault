## Что такое юнит-экономика?

**Юнит-экономика (Unit Economics)** — расчёт прибыли/убытка в расчете на одного клиента. Показывает:
- Есть ли финансовый смысл масштабировать проект
- Где находится точка безубыточности

> [!NOTE] Расчёт юнит-экономики необязателен для защиты ВКР, но может повысить оценку и полезен для реального бизнеса.

---

## Ключевые метрики

### UA (Users / Активные пользователи)
Число привлечённых пользователей — сколько ознакомились с продуктом через маркетинг.

```
UA = количество кликов/звонков в месяц
```

**Пример:** 1000 кликов в месяц или 500 звонков менеджера.

### C1 (Конверсия пользователь → клиент)
Процент пришедших, готовых купить. Обычно **не больше 3%** (берём 2%).

### Buyers (Число клиентов)
```
Buyers = UA × C1
```

**Пример:** UA = 500, C1 = 5% → Buyers = 25 клиентов

### AvP / AvPrice (Average Price — средний чек)
Сумма, которую заплатил клиент за товар/услугу.

Подходы к расчёту:
- **От конкурента:** узнать цену у конкурентов и сделать на 15% дешевле
- **От желаемой прибыли:** через маржу
- **От выгоды клиента:** если нет прямых конкурентов, стоимость = экономия затрат клиента

### COGS (Cost of Goods Sold — себестоимость продаж)
Затраты, которые несём при каждой продаже. Отделить постоянные расходы от переменных.

**Пример для B2B:** премия менеджеру с каждой продажи.

### 1sCOGS (First-sale COGS)
Дополнительные затраты к COGS за первую продажу: пилоты, интеграции, повышенная комиссия агенту.

### APC (Average Payments Count — кол-во платежей)
Среднее число платежей одного клиента за период. Используется при подписочной модели.

> [!WARNING] Нельзя округлять APC!

### ARPC (Average Revenue Per Paying Customer)
Средний доход на одного платящего клиента:

```
ARPC = (AvP - COGS) × APC - 1sCOGS
```

### ARPPU (Average Revenue Per Paying User — LTV)
Доход с одного платящего клиента за всё время отношений:

```
ARPPU = (AvPrice - COGS) × APC - 1sCOGS
```

### ARPU (Average Revenue Per User)
Средний доход на одного пользователя (без учёта маркетинга):

```
ARPU = ARPPU × C1
```

Или:
```
ARPU = ARPC × C1
```

### CPA (Cost Per Acquisition — стоимость привлечения)
Стоимость привлечения одного пользователя:

```
CPA = AC / UA
```
- `AC` — маркетинговый бюджет (постоянные расходы, не зависящие от продаж)

---

## Формула маржинальности

```
МАРЖА = ARPC × C1_клиент - CPA
```
- `ARPC` — средний доход на одного клиента
- `C1_клиент` — число клиентов (Buyers)

> [!IMPORTANT] Если МАРЖА > 0 — бизнес-модель жизнеспособна. Если < 0 — нужно пересматривать цену или снижать затраты.

## Пример расчёта юнит-экономики

### Исходные данные (Месяц 1)

| Параметр | Значение |
|----------|----------|
| UA (звонков в месяц) | 500 |
| C1 (конверсия в продажу) | 5% |
| Buyers (клиентов) | 25 |
| AvPrice (средний чек) | 35 000 руб. |
| COGS (премия с продаж) | 5% |
| 1sCOGS (доп. премия за 1-ю продажу) | 5% |
| APC (платежей на клиента) | 1 |
| AC (зарплата менеджера, постоянная) | 50 000 руб. |

### Расчёт Месяц 1

```
Buyers = UA × C1 = 500 × 5% = 25 клиентов

ARPPU = (AvPrice - COGS) × APC - 1sCOGS
      = (35 000 - 5%) × 1 - 5%
      = (35 000 - 1 750) × 1 - 1 750
      = 31 500 руб.

ARPU = ARPPU × C1 = 31 500 × 5% = 1 575 руб.

CPA = AC / UA = 50 000 / 500 = 100 руб.
```

**Прибыль менеджера:**
- Постоянная: 50 000 руб.
- Переменная (COGS): 1 750 × 25 = 43 750 руб.
- За первую продажу (1sCOGS): 1 750 × 25 = 43 750 руб.
- **Итого менеджеру:** 137 500 руб.

**Доход компании:**
```
(ARPU - CPA) × UA = (1 575 - 100) × 500 = 737 500 руб.
Минус зарплата менеджеру: 737 500 - 137 500 = 600 000 руб. прибыли
```

---

### Расчёт Месяц 2 (клиенты повторяют покупку)

Предполагаем, что все клиенты 1-го месяца совершили повторную покупку:

```
APC = 2 (каждый клиент заплатил 2 раза)

ARPPU = (35 000 - 5%) × 2 - 5% = 64 750 руб.

Зарплата менеджера:
- Постоянная: 50 000 × 2 = 100 000 руб.
- COGS: 1 750 × 25 × 2 = 87 500 руб.
- 1sCOGS (только новые): 1 750 × 25 = 43 750 руб.
- Итого менеджеру: 231 250 руб.

Доход компании:
(ARPPU × C1 - CPA) × UA = (64 750 × 5% - 100) × 500 = 1 568 750 руб.
```
